## 2. 匯率服務實作

- [ ] 2.1 在 `js/exchange-rate.js` 中實作多來源降級鏈：主來源 (`open.er-api.com`) → 備援來源 (jsDelivr CDN 的 `@fawazahmed0/currency-api`) → 本地快取 → 寫死預設值，並將取得的匯率正規化為「1 外幣 = N 台幣」正向格式。
- [ ] 2.2 實作 `localStorage` 中的匯率快取與驗證邏輯（時間戳記效期驗證），並落實請求節流：快取未滿 24 小時不重新請求、單一來源失敗至多重試一次。
- [ ] 2.3 新增 UI 控制項與邏輯，供使用者手動調整與覆寫匯率：手動值需驗證為正數、覆寫後將該貨幣標記為「已鎖定」使自動更新跳過、並提供解除鎖定；同時在使用快取或預設值時於介面顯示「最後更新時間 / 可能過時 / 預設估計值」等狀態提示。

## 3. 計算引擎實作

- [ ] 3.1 實作單位標準化計算函式，將各種重量/容量單位（g, kg, oz, lb, ml, l）統一轉換為公克（g）。
- [ ] 3.2 實作貨幣轉換邏輯，根據當前匯率將所選的外幣價格折算為台幣（TWD）。
- [ ] 3.3 實作輸入表單驗證：拒絕負值/零、空白、非數字字元與非有限/極大值（溢位），全部通過後才觸發計算；計算結果一律四捨五入並固定顯示至小數點後 2 位。

## 4. 比價歷史紀錄管理

- [ ] 4.1 實作歷史比價列表的動態渲染，在網頁上呈現已儲存商品的詳細資訊。
- [ ] 4.2 實作「儲存比價」按鈕事件，將商品的原始輸入（名稱/價格/幣別/重量/單位）、加入時間戳記，以及「當時匯率與當時 TWD/100g」快照存入歷史陣列並同步至 `localStorage`。
- [ ] 4.3 實作比對邏輯：依單位類型將項目分為重量類（每 100g）與容量類（每 100ml）兩組，各組內以「當前匯率」即時重算並找出最低者進行綠色主題高亮（不跨組比較、匯率變動時自動重新排名）；同時於各項目提供「當時價」回顧顯示。
- [ ] 4.4 實作單一商品刪除與一鍵清空所有比價歷史的控制邏輯。

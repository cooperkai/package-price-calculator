## 專案緣由 (Why)

購買商品時，由於包裝容量規格不一（例如 250g、1.5kg、3oz、1lb 等）且可能使用不同的貨幣計價（如美元 USD、日圓 JPY、歐元 EUR），消費者往往難以直接比較價格。
為了做出明智的購物決定，消費者需要一個快速、行動裝置友好的工具，來計算並比較標準化後的商品成本——具體而言，即是將價格根據當天即時匯率換算成新台幣（TWD），並計算每 100 公克的單價。
由於此應用程式將託管於 GitHub Pages 上，它必須是一個輕量、靜態的網頁程式，且能完全在瀏覽器中運行，並在手機收訊不良時支援離線使用。

## 變更內容 (What Changes)

我們將建立一個響應式的單頁漸進式網頁應用程式（PWA），使用純 HTML、現代 CSS 和原生 JavaScript 構建（無複雜的建置系統，可直接部署至 GitHub Pages）。

要實作的核心功能：
- **貨幣與重量換算引擎**：將任何輸入的貨幣和重量單位標準化為台幣與克數，計算對應的每 100g 價格。
- **動態匯率取得**：整合免費、穩定的公開匯率 API 以取得每日匯率，並具備自動本地快取（localStorage）與手動覆寫功能。
- **商品比價歷史列表**：在比價表格中儲存已計算的商品列表，並自動高亮最划算的商品。
- **離線與行動優先 (PWA)**：實作 Service Worker 以提供離線功能，確保在地下室或訊號不佳的超市中仍能正常運行。

## 系統能力 (Capabilities)

### 新增系統能力
- `unit-price-calculator`：核心計算邏輯，將各種輸入單位（g, kg, oz, lb, ml, l）與價格換算為台幣每 100g 單價。
- `exchange-rate-service`：動態從公開 API 取得每日匯率、將匯率快取於 localStorage，並提供手動調整匯率的設定。
- `comparison-history`：互動式歷史紀錄，允許使用者新增、比對、重新命名或清除多個商品，以便快速找出最便宜的選擇。
- `pwa-offline-support`：結合 Service Worker 與 manifest.json 的漸進式網頁應用程式設定，支援離線運行與新增至主畫面安裝。

### 修改系統能力
*(無。此為全新的專案。)*

## 專案影響 (Impact)

- **新增資源**：
  - `index.html`：應用程式的主結構網頁。
  - `css/style.css`：現代化、響應式的 CSS 樣式，包含深色主題、流暢過渡效果與清晰字型。
  - `js/app.js`：核心邏輯，包含計算、歷史紀錄管理與 UI 事件處理。
  - `js/exchange-rate.js`：匯率取得與快取管理模組。
  - `sw.js`：用於離線快取的 Service Worker 腳本。
  - `manifest.json`：支援 PWA 安裝的設定檔。
- **代管方式**：完全設計為無須建置步驟的靜態網頁，極易託管於 GitHub Pages。

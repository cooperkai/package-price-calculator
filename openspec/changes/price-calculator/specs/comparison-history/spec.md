## ADDED Requirements

### Requirement: 儲存商品至比價列表
系統應（SHALL）允許使用者將當前的計算結果儲存至列表中。
列表中的每個項目應包含：
- 自訂商品名稱（若為空則預設為 "項目 #"）
- 原始輸入的價格、幣別、重量/容量與單位（此為永久不變的紀錄）
- 加入當下的時間戳記
- 加入當下所用的匯率，以及當下計算出的台幣每 100 克（TWD / 100g）單價（即「當時價」快照，僅供回顧顯示，不作為排名比較依據）

#### Scenario: 新增商品至列表
- **WHEN** 使用者在合法的計算結果下點選「新增至比價列表」按鈕
- **THEN** 系統應將商品資訊加入至比價表格中，並將列表同步儲存至 `localStorage`

### Requirement: 依單位類型分組、以當前匯率即時重算並高亮最划算的商品
系統應（SHALL）依「單位類型」將項目自動分為兩組——**重量類**（g、kg、oz、lb，以每 100g 比較）與**容量類**（ml、l，以每 100ml 比較）——並在每一組**內部**各自以「當前匯率」即時重算台幣單價（而非使用儲存的當時快照），找出該組最低者進行高亮。系統不應（SHALL NOT）跨組比較（例如將重量類商品與容量類商品放在一起評選最划算）。

#### Scenario: 同時包含重量類與容量類商品
- **WHEN** 比價列表同時含有重量類（如白米 g）與容量類（如洗髮精 ml）商品時
- **THEN** 系統應分別於重量類與容量類各自找出每 100g／每 100ml 台幣單價最低的項目，各自加上「最划算」標籤與醒目背景（例如淺綠色主題），兩組互不影響

#### Scenario: 單一分組內的多個商品
- **WHEN** 某一單位類型分組內包含兩個或以上的項目時
- **THEN** 系統應以當前匯率重算該組每個項目的單價，找出最低者進行高亮

#### Scenario: 匯率更新後重新排名
- **WHEN** 匯率因每日自動更新或使用者手動調整而改變時
- **THEN** 系統應以新匯率重算所有項目，並在各自分組內重新評估「最划算」高亮，使排名即時反映當前匯率

### Requirement: 顯示當時價快照
系統應（SHALL）在每個項目上提供「當時價」資訊，讓使用者回顧該商品加入當下的台幣單價與所用匯率。

#### Scenario: 檢視項目的當時價
- **WHEN** 使用者檢視某個已儲存的項目
- **THEN** 系統應顯示該項目「加入當時」的每 100g／每 100ml 台幣單價與當時匯率，與「以當前匯率重算」的單價並列或可展開查看

### Requirement: 管理比價列表
系統應（SHALL）提供刪除單一項目或清空整個列表的控制項。

#### Scenario: 刪除單一項目
- **WHEN** 使用者點選比價列表中某個商品旁的刪除按鈕
- **THEN** 系統應將該商品從列表中移除，並重新評估與更新「最划算」高亮項目

#### Scenario: 清空所有項目
- **WHEN** 使用者點選「清空所有紀錄」按鈕
- **THEN** 系統應將所有項目從列表中刪除，並清空 UI 顯示與 `localStorage` 內的暫存資料
